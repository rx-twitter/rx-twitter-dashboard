## 調査 1: oslo の実際のエンコーディング仕様

**問い**: `oslo/encoding` の `base32.encode()` はパディングあり・なしどちらか？

**実測結果**（Node.js v実行済み）:

```
入力: new Uint8Array([0x66])           → "MY======"  (パディングあり)
入力: new Uint8Array(20) (乱数)        → "XXXXX...X" (32文字、パディングなし)
入力の 20 % 5 === 0                    → 40ビットブロックが揃う → パディント不要
```

**決定事項**: oslo は RFC 4648 Base32（パディングあり）だが、`generateSessionId()` が使用する **20バイト入力では必ずパディングなし 32文字** が出力される。

**根拠**: 160ビット ÷ 5ビット/文字 = 32文字ちょうど。余りビットが発生しないため `=` パディントは付与されない。自前実装は「パディングなし RFC 4648」として実装すれば20バイト入力で完全互換となる。

**検討した代替案**: パディングあり実装で oslo に完全準拠 → 不要（セッション ID 生成の用途ではパディングが発生しないため、複雑性が増すだけ）。

## 調査 2: oslo 以外の代替パッケージの評価

**問い**: `@oslojs/encoding` 等の後継パッケージは使えるか？

| パッケージ         | 最終更新          | 週 DL | 評価                    |
| ------------------ | ----------------- | ----- | ----------------------- |
| `oslo`             | アーカイブ済み    | -     | ❌ 不使用               |
| `@oslojs/encoding` | 2年前（2024年頃） | 少    | ⚠️ 将来同様にリスクあり |
| `hi-base32`        | アクティブ        | 中    | △ 依存追加が惜しい      |
| 自前実装           | N/A               | N/A   | ✅ 採用                 |

**決定事項**: **自前実装を採用する**。

**根拠**:

- Base32（RFC 4648）は仕様が完全に固定されたアルゴリズム。暗号プリミティブと違い脆弱性リスクがなく、実装が変わる理由がない
- エンコードのみ（デコード不要）であれば実装は約10行に収まる
- 外部依存をゼロにすることで `@oslojs/encoding` 同様のリスクを根本回避できる

**検討した代替案**: `node:crypto` の `Buffer.from(bytes).toString('base64url')` → Base32 ではないため既存セッション ID との非互換が生じる。採用不可。

## 調査 3: RFC 4648 テストベクターの確認

**問い**: 自前実装の正確性を担保するテストベクターは何か？

**RFC 4648 §10 公式テストベクター（Base32）**:

| 入力（ASCII） | 期待出力（パディングなし） |
| ------------- | -------------------------- |
| `""`          | `""`                       |
| `"f"`         | `"MY"`                     |
| `"fo"`        | `"MZXQ"`                   |
| `"foo"`       | `"MZXW6"`                  |
| `"foob"`      | `"MZXW6YQ"`                |
| `"fooba"`     | `"MZXW6YTB"`               |
| `"foobar"`    | `"MZXW6YTBOI"`             |

**追加エッジケースベクター**:

| 入力                            | 期待出力                                                    | 確認事項     |
| ------------------------------- | ----------------------------------------------------------- | ------------ |
| `new Uint8Array(20).fill(0x00)` | `"AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA"` → 実際は32文字の `A` | 全ゼロバイト |
| `new Uint8Array(20).fill(0xFF)` | `"77777777777777777777777777777777"`                        | 全 FF バイト |
| `new Uint8Array(20)` (乱数)     | 長さ32、`[A-Z2-7]` のみ                                     | 形式検証     |

**決定事項**: `encoding.test.ts` に上記ベクターを全件含める。
